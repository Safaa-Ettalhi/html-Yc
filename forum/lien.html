Introduction à HTML5

Dans cet article HTML5 est la cinquième révision et la dernière version de la norme HTML. Il offre de nouvelles fonctionnalités qui fournissent non seulement la prise en charge des médias enrichis, mais améliorent également la prise en charge de la création d’applications Web capables d’interagir avec les utilisateurs, leurs données locales et les serveurs plus facilement et plus efficacement qu’auparavant.
Certaines fonctionnalités HTML5 ne sont pas prises en charge par certains navigateurs. Cependant, Gecko, et par extension, Firefox, a un très bon support pour HTML5, et le travail se poursuit pour prendre en charge davantage de ses fonctionnalités.
Gecko a commencé à prendre en charge certaines fonctionnalités HTML5 dans la version 1.8.1.
Vous pouvez trouver une liste de toutes les fonctionnalités HTML5 que Gecko prend actuellement en charge sur la page principale HTML5. Pour obtenir des informations détaillées sur la prise en charge des fonctionnalités HTML5 par plusieurs navigateurs, consultez le site Web de CanIUse.

Déclarer que le document contient un balisage HTML5 avec le doctype HTML5 Le doctype pour HTML5 est très simple. Pour indiquer que votre contenu HTML utilise HTML5, il vous suffit d’utiliser :
< !DOCTYPE html> Ce faisant, même les navigateurs qui ne prennent pas actuellement en charge HTML5 passeront en mode standard, ce qui signifie qu’ils interpréteront les parties établies de longue date de HTML d’une manière conforme à HTML5 tout en ignorant les nouvelles fonctionnalités de HTML5 qu’ils ne prennent pas en charge.
C’est beaucoup plus simple que les anciens doctypes, et plus court, ce qui facilite la mémorisation et réduit la quantité d’octets à télécharger.
Déclaration du jeu de caractères avec le < meta charset> La première chose à faire sur une page est généralement d’indiquer le jeu de caractères utilisé.
Dans les versions précédentes de HTML, cela se faisait à l’aide d’un élément < méta> très complexe. Maintenant, c’est très simple :
< meta charset="UTF-8"> Placez-le juste après votre < head>, car certains navigateurs relancent l’analyse d’un document HTML si le jeu de caractères déclaré est différent de ce qu’ils avaient prévu. De plus, si vous n’utilisez pas actuellement UTF-8, il est recommandé de l’utiliser dans vos pages Web, car il simplifie la gestion des caractères dans les documents utilisant différents scripts.
Notez que HTML5 limite les jeux de caractères à ceux compatibles avec ASCII et utilisant au moins 8 bits. Cela a été fait pour renforcer la sécurité et prévenir certains types d’attaques.
Utilisation du nouvel analyseur HTML5
Les règles d’analyse de HTML5, qui analysent la signification du balisage, ont été définies plus précisément en HTML5. Jusqu’à l’introduction de HTML5, seule la signification du balisage valide était définie, ce qui signifie que dès qu’une petite erreur était commise dans le balisage (la plupart des sites Web en ont au moins une), le comportement n’était pas défini. Essentiellement, cela signifiait que tous les navigateurs se comportaient différemment, ce qui n’est plus le cas aujourd’hui. Désormais, face aux erreurs de balisage, tous les navigateurs conformes doivent se comporter exactement de la même manière.

Cette exigence aide beaucoup les développeurs Web. S’il est vrai que tous les navigateurs modernes utilisent désormais ces règles d’analyse HTML5, les navigateurs non conformes à HTML5 sont toujours utilisés par certains. Gardez à l’esprit qu’il est toujours fortement recommandé d’écrire un balisage valide, car ce code est plus facile à lire et à maintenir, et il réduit considérablement la proéminence des incompatibilités qui existent dans divers navigateurs plus anciens.
Ne vous inquiétez pas, vous n’avez rien à changer sur votre site Web. Les développeurs des navigateurs web ont tout fait pour vous !

Css

CSS (pour Cascading Style Sheets en anglais), soit feuilles de style en cascade, est un langage de feuille de style utilisé pour décrire la présentation d'un document écrit en HTML ou XML (y compris les dialects XML que sont SVG, MathML, ou XHTML).
CSS décrit la façon dont les éléments doivent être affichés à l'écran, sur papier, à l'oral ou sur d'autres médias.
CSS est l'un des langages principaux du Web ouvert et a été standardisé par le W3C.
Auparavant, le développement des différentes parties de la spécification CSS était réalisé de façon synchrone, permettant d'avoir une version pour l'ensemble de la recommandation.
Vous avez donc pu entendre parler de CSS1, CSS2.1, voire CSS3.
Toutefois, il n'y aura pas de version CSS4 ou d'autres version globale numérotée de CSS.
En effet, après CSS 2.1, la portée de la spécification a augmenté significativement et les avancées sur les différents modules CSS ont commencé à diverger, et il est alors devenu plus efficace de développer et diffuser les recommandations par module.
Plutôt que d'avoir une version globale de la spécification CSS, le W3C collecte désormais de façon périodique un instantané du dernier état stable de la spécification CSS et de l'avancée de chaque module.
Les modules de spécification CSS ont un numéro de version, appelé niveau, comme le module de spécification CSS sur les couleurs de niveau 5
